# Story 4.3: Weryfikacja danych na produkcji — test end-to-end

Status: in-progress

## Story

Jako **operator**,
chcę **zweryfikować dane na produkcji (test end-to-end), aby potwierdzić gotowość aplikacji na pilota z pierwszym dostawcą**,
aby **mieć pewność, że dane fixtures działają poprawnie w środowisku produkcyjnym**.

## Acceptance Criteria

1. **Aplikacja uruchamia się na produkcji** (Vercel) bez błędów.
2. **Tenant `swieze-z-kaszub`** jest dostępny publicznie (`/[tenant]`).
3. **Katalog produktów** wyświetla się poprawnie ze zdjęciami (jeśli dostępne) i cenami.
4. **Checkout** działa z wybranym tenantem (logowanie, koszyk, zamówienie).
5. **Test manualny** — przejście entire flow od przeglądania katalogu do złożenia zamówienia.

## Tasks / Subtasks

- [ ] **Task 1 — Weryfikacja deployu** (AC: 1)
  - [ ] Sprawdź status deployu na Vercel
  - [ ] Sprawdź logi — brak błędów
- [ ] **Task 2 — Test tenanta** (AC: 2, 3)
  - [ ] Otwórz `/swieze-z-kaszub` w przeglądarce
  - [ ] Sprawdź listę produktów, ceny, opisy
- [ ] **Task 3 — Test checkoutu** (AC: 4, 5)
  - [ ] Zaloguj się jako klient
  - [ ] Dodaj produkt do koszyka
  - [ ] Wybierz adres dostawy
  - [ ] Złóż zamówienie
  - [ ] Sprawdź e-mail potwierdzający
