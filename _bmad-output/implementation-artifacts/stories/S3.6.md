# S3.6 — Filtr po kategorii na katalogu (server-side)

Status: ☐ TODO · ready-for-dev · EPIC-3 · utworzono 2026-06-21

## Historyjka

Jako **klient** chcę **filtrować katalog `/[tenant]` po kategorii**, aby **szybciej znaleźć produkty** (D4).

## Kryteria akceptacji (z sprint-3.md)

- Katalog `/[tenant]?kategoria=<slug>` filtruje **server-side** w `getCatalog` (where `tenant` + dopasowanie kategorii w relacji `hasMany`).
- Produkt przypisany do wielu kategorii pojawia się w **każdej** ze swoich kategorii (D4).
- Bez parametru — wszystkie produkty tenanta.
- Lista kategorii w UI pochodzi tylko z tego tenanta.
- Nieznana/cudza kategoria → pusto/ignorowana, **nigdy cross-tenant**.

## Zależności

- S3.4, S3.3.

## Uwagi implementacyjne

- **Read-path (R-S3.5):** `getCatalog` czyta `overrideAccess: true` + ręczny `where` — filtr kategorii **MUSI** zawsze dokładać `tenant` do `where`, inaczej przeciek.
- Param URL `?kategoria=` → slug kategorii; lista kategorii w UI z `where { tenant }`.

## Plan weryfikacji

- Filtr po kategorii zwraca tylko produkty tenanta z tej kategorii; produkt wielokategoryjny widoczny w każdej; nieznana/cudza kategoria → pusto, nigdy cross-tenant.
