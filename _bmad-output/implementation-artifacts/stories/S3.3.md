# S3.3 — Render zdjęcia (katalog + szczegół zamówienia) z fallbackiem

Status: ☐ TODO · ready-for-dev · EPIC-3 · utworzono 2026-06-21

## Historyjka

Jako **klient** chcę **widzieć zdjęcie na katalogu `/[tenant]` i w szczegółach zamówienia**, aby **wiedzieć co kupuję** (D3).

## Kryteria akceptacji (z sprint-3.md)

- Render wybiera zdjęcie wg fallbacku **wariant → produkt → placeholder** (D3).
- Karta produktu: `next/image` (rozmiary responsywne, `alt` z mediów); placeholder gdy brak na obu poziomach.
- Szczegół zamówienia (panel + front klienta): miniatura pozycji = zdjęcie **wybranego wariantu**, fallback do produktu.
- Render nie łamie istniejącego katalogu/koszyka.

## Zależności

- S3.2.

## Uwagi implementacyjne

- **Logika fallbacku w jednym helperze** (reuse, testowalność, R-S3.6): `resolveProductImage(variant, product)`.
- **Wydajność (NFR8, R-S3.3):** `next/image` + warianty `sharp` + lazy-load; mierzyć LCP listy produktów.
- Front czyta publicznie: `overrideAccess: true` + ręczny `where { tenant }` (architektura §3) — przy renderze nie przeciekać mediów cross-tenant.

## Plan weryfikacji

- Karta produktu i miniatura pozycji pokazują właściwe zdjęcie wg fallbacku; placeholder gdy brak; LCP listy pod kontrolą; katalog/koszyk działają.
