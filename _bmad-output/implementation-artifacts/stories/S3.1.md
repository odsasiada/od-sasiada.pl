## Dev Notes

### Co to za historia — stan faktyczny (UPDATE, nie greenfield)
Baseline mediów **już istnieje** z SPIKE-S3 — NIE pisać kolekcji ani adaptera od nowa. Realna pozostała praca w tej historii to dwie rzeczy:
1. **Test izolacji** (AC3) — wykonalny bez tokenu, zamyka dziurę „model gotowy, brak testu".
2. **Weryfikacja realnego bloba + URL** (AC4/AC5) — **zablokowana** na `BLOB_READ_WRITE_TOKEN` (akcja użytkownika).
Reszta (kolekcja, sharp, alt, multi-tenant, adapter warunkowy, ENV) jest gotowa i zielona.

### Stan obecny plików (przeczytane — NIE psuć)
- **`src/collections/Media.ts`**: `slug: 'media'`, `upload.imageSizes` (`thumbnail` 400×400, `card` 768w, `hero` 1200w), `mimeTypes: ['image/*']`, pole `alt` (wymagane, label PL „Tekst alternatywny"), `access` panel-only (`isAdmin` na create/read/update/delete), `admin: { group: 'Shop', useAsTitle: 'alt' }`. **`read` celowo NIE publiczny** — front czyta przez `overrideAccess: true` + ręczny `where { tenant }` w S3.3 (R-S3.2). Komentarz w pliku to dokumentuje.
- **`src/payload.config.ts`**: `Media` w `collections: [Users, Tenants, Customers, Media]` (`#52`); `media: {}` w `multiTenantPlugin.collections` (`#170`); `vercelBlobStorage({ collections: { media: true }, token: env.BLOB_READ_WRITE_TOKEN })` dodany do `plugins` **warunkowo** `...(env.BLOB_READ_WRITE_TOKEN ? [...] : [])` (`#187`). Bez tokenu adapter nieaktywny → dysk lokalny (dev), app wstaje.
- **`src/env.ts`**: `BLOB_READ_WRITE_TOKEN` jako **opcjonalny** (`z.string().min(1).optional()`). Token czytany TYLKO przez `env` (nie `process.env` bezpośrednio).

### Izolacja (NFR1 / R-S3.2) — model i co potwierdzić
- **Panel/zapis:** `multiTenantPlugin` stempluje `tenant_id` i filtruje wiersze do tenanta admina; `platform-admin` (`userHasAccessToAllTenants`) widzi wszystko. Wzór jak `orders`/`carts` (SPIKE-A potwierdził, że plugin pilnuje — własny `access` po tenant zbędny).
- **`read` nie-publiczny:** zamierzone — publiczny odczyt mediów szedłby przez front z `overrideAccess: true` + `where { tenant }` (S3.3), nie przez publiczny `access.read`. To chroni przed wyciekiem listy cross-tenant.
- **Otwarte (AC5):** sam access Payloada NIE chroni pliku, jeśli Vercel Blob serwuje go z publicznej, przewidywalnej domeny URL. Po odblokowaniu tokenu trzeba empirycznie ustalić model serwowania i — jeśli URL publiczny — oprzeć izolację na nieprzewidywalności ścieżki + braku listowania, oraz to udokumentować (architecture §8).

### Blokada tokenu — co i jak (akcja użytkownika)
`BLOB_READ_WRITE_TOKEN` pochodzi z integracji **Vercel Marketplace** (Blob). Bez niego AC4/AC5 nie domknąć (adapter jest wpięty, ale realny upload/serwowanie niewykonalne). Kroki: podłącz integrację Blob w projekcie Vercel → `vercel env pull` (zaciąga do `.env.local`) albo wpisz token ręcznie w `.env.local`. **NIE commitować tokenu.** Na Vercel ENV wstrzykiwany automatycznie przez integrację. (Jeśli chcesz wykonać interaktywnie w tej sesji: `! vercel env pull`.)

### Środowisko / workflow
- **Postgres = Docker** (`od-sasiada-pg`, `postgres:17`, port 5432, baza `od_sasiada`) — NIE Homebrew. Daemon padł: `open -a Docker` → `docker info` → `docker start od-sasiada-pg`.
- **Po zmianie kolekcji `pnpm payload:types`** (i `payload:importMap` przy komponentach admina). Nie edytować `payload-types.ts`/`importMap.js`.
- **Deploy = Vercel**, host efemeryczny (brak trwałego dysku) → dlatego Blob, nie dysk lokalny w produkcji (D1).
- **Biome** (single quotes, bez średników, `organizeImports` ON) — `pnpm lint:fix`.
- **pnpm install policy:** `pnpm-workspace.yaml` ma `minimumReleaseAgeExclude: ['@img/sharp-*', 'happy-dom']` (z SPIKE-S3) — nie ruszać.
