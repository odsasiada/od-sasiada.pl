# Story 2.6: PL-izacja i wzbogacenie maili (slot dostawy + kontakt dostawcy)

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

Jako **operator platformy**,
chcę **treści maili (potwierdzenie + zmiana statusu) po polsku oraz z terminem dostawy i telefonem kontaktowym dostawcy**,
aby **komunikacja z klientem była po polsku i kompletna** (domknięcie pętli „wiem co się dzieje").

## Acceptance Criteria

1. **Treść maila potwierdzającego (`sendOrderConfirmation`) po polsku** — nagłówek, opis, etykiety (pozycje, suma, adres), temat. Kwoty nadal w formacie `X,XX zł`.
2. **Treść maila statusowego (`sendStatusChange`) po polsku** — nagłówek, zdanie o zmianie statusu, temat; etykieta statusu z PL `ORDER_STATUS_LABELS`.
3. **Etykiety statusów PL** — `ORDER_STATUS_LABELS` w `src/ecommerce/order-status.ts` po polsku (dziś po ang.). Skutek spójny WSZĘDZIE, gdzie używane: opcje pola `status` w panelu, strony frontu (`moje-zamowienia`), komunikat błędu przejścia, maile.
4. **Mail potwierdzający zawiera termin dostawy** ze snapshotu `deliverySlot` zamówienia (S2.4) — gdy zamówienie ma slot; gdy nie ma (O8) → sekcja pominięta, bez błędu.
5. **Mail potwierdzający zawiera telefon kontaktowy dostawcy** (`tenant.settings.contactPhone`) — gdy ustawiony; gdy brak → sekcja pominięta.
6. **Best-effort zachowane (NFR4)** — błąd SMTP/odczytu logowany, NIE blokuje operacji zapisu zamówienia; brak `customerEmail` → brak wysyłki (zachowany guard).
7. **Testy zaktualizowane i zielone** — `tests/unit/order-emails.test.ts` (asercje treści ang. → PL + slot/telefon), ewentualnie `order-status.test.ts`; `pnpm lint` + `pnpm test` bez regresji; `pnpm dev` startuje.

### Dane do treści — skąd je wziąć
- **Slot:** ze snapshotu `doc.deliverySlot` (S2.4: `date`, `windowStart`, `windowEnd`, `label`). Preferuj gotowe `label` (jedno źródło formatowania, `formatSlotLabel` z S2.4); fallback złóż z `date` + `windowStart–windowEnd`. Render warunkowo (O8: brak slotu → brak sekcji).
- **Telefon dostawcy:** `tenant.settings.contactPhone`. Builder ma `payload` → `payload.findByID({ collection: 'tenants', id: tenantIdOf(doc), depth: 0, disableErrors: true })`. `doc.tenant` bywa `number | { id }` — znormalizuj (wzór `idOf` w `actions.ts`/`cart.ts`). Best-effort: błąd/brak → pomiń sekcję, NIE wywracaj maila. (Wzór mapowania `contactPhone` już jest w `getTenantBySlug`, `src/lib/shop.ts:25`.)

### NFR4 — best effort
