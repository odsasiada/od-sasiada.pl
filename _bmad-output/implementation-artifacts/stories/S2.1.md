## Story

Jako **dostawca**,
chcę **skonfigurować tygodniowe okna dostawy + cutoff + limit miejsc w dedykowanej, multi-tenantowej kolekcji `DeliverySlots`**,
aby **klient w checkoucie wybierał wyłącznie realne terminy dostawy** (decyzje O1–O4, EPIC-2).

## Acceptance Criteria

1. **Nowa kolekcja `DeliverySlots`** istnieje (`src/collections/DeliverySlots.ts`, `CollectionConfig`, slug **`delivery-slots`**) i jest wpięta do `collections: [...]` w `payload.config.ts` ORAZ do `multiTenantPlugin.collections` (klucz `delivery-slots`) → izolacja tenant gratis (dostawca widzi/edytuje tylko swoje sloty; `platform-admin` widzi wszystkie).
2. **Pola zgodne 1:1 z kontraktem SPIKE-S2** (typ `DeliverySlot` w `src/lib/delivery-slots.ts`):
   - `weekday` — `number`, integer 0–6 (**0 = niedziela … 6 = sobota**, zgodnie z JS `getDay()`),
   - `windowStart` — `text`, wall-clock `HH:mm` (Europe/Warsaw),
   - `windowEnd` — `text`, `HH:mm`,
   - `cutoffDaysBefore` — `number`, integer ≥ 0,
   - `cutoffTime` — `text`, `HH:mm`,
   - `capacity` — `number`, integer ≥ 0;
   - relacja `tenant` dodawana automatycznie przez plugin (**NIE definiować ręcznie**).
3. **Pola i etykiety po polsku** (`label` PL, `admin.description` po polsku). Kod/identyfikatory po angielsku.
4. **Walidacja sensowności przy zapisie** (panel + Local API):
   - `windowEnd > windowStart` (porównanie minut z `HH:mm`),
   - `windowStart` / `windowEnd` / `cutoffTime` w formacie `^\d{2}:\d{2}$` z poprawnym zakresem (00–23:00–59),
   - `weekday` ∈ 0..6 (integer),
   - `cutoffDaysBefore` ≥ 0 (integer),
   - `capacity` ≥ 0 (integer).
5. **`reservedCount` NIE jest polem kolekcji** — obłożenie liczone z `orders` (S2.7). `DeliverySlots` trzyma wyłącznie config.
6. **Brak slotów u tenanta = feature wyłączony (O8)** — kolekcja może być pusta; nie ma żadnego seeda wymuszającego sloty.
7. **`pnpm payload:types` przechodzi** (typ `DeliverySlot` w `src/payload-types.ts`), **`pnpm test` zielone bez regresji** na live Docker Postgres (`od-sasiada-pg`), **`pnpm dev` startuje**.

### Kontrakt kształtu pól — JUŻ ZABLOKOWANY (SPIKE-S2)
Pola muszą odwzorować typ `DeliverySlot` z `src/lib/delivery-slots.ts:29`. To źródło prawdy — nie wymyślaj innych nazw/typów. Mapowanie:

| Pole kolekcji | Typ Payload | Uwagi |
|---|---|---|
| `weekday` | `number` | integer 0–6; **number, NIE string** (locked SPIKE: `computeAvailableSlots` porównuje wprost z `Intl`/`getUTCDay`, zero mapowania w hot-path). Etykiety dni w `admin.description`. |
| `windowStart` | `text` | `HH:mm` wall-clock Europe/Warsaw |
| `windowEnd` | `text` | `HH:mm`; `> windowStart` |
| `cutoffDaysBefore` | `number` | integer ≥ 0; „na jutro" = `1`, same-day = `0` |
| `cutoffTime` | `text` | `HH:mm`; godzina graniczna na dniu `dzień_dostawy − cutoffDaysBefore` |
| `capacity` | `number` | integer ≥ 0; limit aktywnych zamówień na JEDNO wystąpienie slotu |
| `tenant` | (plugin) | **NIE definiować** — dodaje `multiTenantPlugin` |

⚠️ **`reservedCount` ani `id` NIE są polami do zdefiniowania.** `id` jest automatyczne; `reservedCount` to argument wejściowy czystej funkcji liczony z `orders` w S2.7. `DeliverySlots` = tylko config (brak driftu — jedno źródło prawdy obłożenia to zamówienia).

### Decyzja: `weekday` jako `number` (nie `select`)
SPIKE-S2 zablokował reprezentację `number 0–6`. Payload `select` przechowuje wartości jako **string**, co wymusiłoby `Number(weekday)` w ścieżce odczytu i łamie locked-decision „zero mapowania". Użyj `type: 'number'` z `min: 0, max: 6`, walidacją integer i `admin.description: '0 = niedziela, 1 = poniedziałek … 6 = sobota'`. (Ładniejszy dropdown z PL-nazwami to potencjalny custom admin component — Open Question Q1, nie blokujące.)

### Walidacja (AC#4) — field-level `validate`
Payload v3 `validate: (value, { siblingData }) => true | string` (komunikat błędu = string PL). Wzór już w repo: walidacja `postalCode` w `payload.config.ts:104` (regex → string PL).
- Czas `HH:mm`: regex `^([01]\d|2[0-3]):[0-5]\d$` (ostrzejszy niż `^\d{2}:\d{2}$` z compute — łapie 99:99 przy zapisie; czysta funkcja i tak ma własny guard).
- `windowEnd > windowStart`: w `validate` pola `windowEnd` sięgnij po `siblingData.windowStart`, sparsuj oba do minut, porównaj. Komunikat PL: `'Koniec okna musi być późniejszy niż początek.'`
- `number`-y: `min`/`max` + `validate` na integer (`Number.isInteger`) gdzie trzeba; `capacity`/`cutoffDaysBefore` ≥ 0; `weekday` 0–6.
- Komunikaty błędów po polsku (i18n `pl`).

### Wpięcie multi-tenant — wzorzec do naśladowania
`Media` to dokładny precedens: kolekcja niestandardowa (nie-ecommerce) wpięta i w `collections: [...]` (`payload.config.ts:52`) i w `multiTenantPlugin.collections` jako `media: {}` (`payload.config.ts:170`). Powtórz 1:1 dla `delivery-slots`. Plugin stempluje `tenant_id`, ogranicza wiersze w panelu do tenanta admina, `platform-admin` (`userHasAccessToAllTenants`) widzi wszystko. **Kolejność pluginów `[ecommerce, multiTenant]` zostaje bez zmian.**
